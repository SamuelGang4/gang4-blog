<SectionHeader number="01" title="Ce qu'on va faire (et pourquoi ça change quelque chose)" />

Voici le scénario concret.

Tu ouvres Claude Desktop. Tu tapes : "Donne-moi le CPA moyen sur mes campagnes Meta des 14 derniers jours, et dis-moi lesquelles sont en learning phase." Claude va directement chercher l'information dans ton compte Meta, analyse et te répond. Pas d'export CSV, pas de connexion à Ads Manager, pas de dashboard à ouvrir.

C'est ce que le Meta Ads MCP rend possible depuis le 29 avril 2026.

MCP signifie Model Context Protocol. C'est un standard ouvert créé par Anthropic en novembre 2024, qui sert de pont entre un agent IA (Claude, ChatGPT) et un service externe (ton compte Meta). Meta a sorti son propre serveur MCP officiel en open beta, ce qui veut dire que tout annonceur peut maintenant connecter son compte Meta à Claude Desktop sans passer par la Marketing API (qui nécessitait un app review développeur et plusieurs jours de validation).

**Pourquoi c'est particulièrement utile pour toi.**

Tu es Traffic Manager DNVB. Tu passes une partie non négligeable de ta journée à naviguer entre onglets Ads Manager, à faire des exports, à compiler du reporting. Le MCP ne change pas ta stratégie. Il change le temps que tu passes à faire le reporting et les analyses de surface, pour que tu investisses ce temps sur les décisions qui comptent : le créatif, l'audience, le budget.

<SectionHeader number="03" title="Étape 1 : installer Claude Desktop" eyebrow="Installation" />

Si tu as déjà Claude Desktop installé, saute cette étape et va directement à l'étape 2. Sinon :

**Sur Mac :**

1. Rends-toi sur `claude.com/download`
2. Clique sur "Download for macOS"
3. Ouvre le fichier `.dmg` téléchargé
4. Fais glisser l'icône Claude dans ton dossier Applications
5. Ouvre Claude depuis Applications (ou Spotlight avec CMD + Espace puis "Claude")
6. Connecte-toi avec ton compte Anthropic (ou crée-en un si tu n'en as pas)

**Sur Windows :**

1. Rends-toi sur `claude.com/download`
2. Clique sur "Download for Windows"
3. Exécute le fichier `.exe` téléchargé
4. Suis l'assistant d'installation (next, next, install)
5. Lance Claude depuis le Bureau ou le menu Démarrer
6. Connecte-toi avec ton compte Anthropic

**Vérification rapide.**

Une fois Claude ouvert et connecté, tu devrais voir l'interface de chat habituelle. Dans le coin inférieur droit de la zone de saisie, il n'y a pas encore d'icône de marteau (ce sera après la configuration). C'est normal à cette étape.

<Callout variant="tip" title="Toujours maintenir Claude Desktop à jour">
Pour que les connecteurs MCP fonctionnent correctement, tu dois être sur la dernière version de Claude Desktop. Clique sur le menu Claude (Mac : dans la barre de menus en haut, pas dans la fenêtre / Windows : dans la barre système) puis "Check for Updates...". Un Claude Desktop trop ancien peut ne pas avoir le support connecteurs activé.
</Callout>

<SectionHeader number="04" title="Étape 2 : ouvrir le fichier de configuration" eyebrow="Configuration" />

C'est la partie qui fait peur aux débutants. En réalité, tu vas juste modifier un fichier texte avec un format précis.

**Comment accéder aux paramètres Developer :**

Sur Mac, clique sur le menu **Claude** dans la barre de menus système en haut (pas la roue crantée dans la fenêtre Claude). Sélectionne **Settings**. Dans la fenêtre Paramètres qui s'ouvre, clique sur l'onglet **Developer** dans le menu de gauche. Tu verras un bouton **Edit Config**.

Sur Windows, clique sur l'icône Claude dans la barre des tâches (zone de notification en bas à droite). Sélectionne **Settings**. Même navigation ensuite : onglet **Developer** puis **Edit Config**.

Clique sur **Edit Config**. Cela ouvre ou crée le fichier de configuration Claude Desktop dans ton éditeur de texte par défaut (souvent Bloc-notes sur Windows ou TextEdit sur Mac).

**Où se trouve le fichier :**

- Mac : `~/Library/Application Support/Claude/claude_desktop_config.json`
- Windows : `%APPDATA%\Claude\claude_desktop_config.json`

Si le fichier n'existait pas encore, Claude Desktop l'a créé vide. Si tu as déjà des MCP servers configurés, tu verras un JSON existant.

<Callout variant="tip" title="Ouvrir le fichier de config manuellement sur Mac">
Si tu ne trouves pas le chemin via le menu Claude, tu peux ouvrir le fichier directement. Dans le Finder, appuie sur CMD + Shift + G, colle ce chemin : `~/Library/Application Support/Claude/` et appuie sur Entrée. Tu verras le fichier `claude_desktop_config.json`. Ouvre-le avec TextEdit (clic droit puis "Ouvrir avec").
</Callout>

<SectionHeader number="05" title="Étape 3 : ajouter le connecteur Meta Ads MCP" eyebrow="Le bloc JSON" />

Tu as maintenant le fichier de configuration ouvert. C'est ici que tu ajoutes le serveur MCP Meta.

Selon la documentation officielle Meta Business Help Center, **l'URL du serveur MCP est universelle pour tous les agents IA supportés** (Claude, ChatGPT, Perplexity) :

> `https://mcp.facebook.com/ads`

C'est cette URL hosted que tu ajoutes dans ta config. Pas de package npm à installer, pas de chemin local à renseigner.

**Si le fichier est vide**, colle ce bloc complet :

```json
{
  "mcpServers": {
    "meta-ads": {
      "url": "https://mcp.facebook.com/ads"
    }
  }
}
```

**Si le fichier contient déjà d'autres MCP servers** (par exemple un filesystem server installé en local via npx), tu dois ajouter Meta Ads à l'intérieur de l'objet `mcpServers` existant, séparé par une virgule :

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/Users/ton-nom/Desktop"]
    },
    "meta-ads": {
      "url": "https://mcp.facebook.com/ads"
    }
  }
}
```

Remplace `ton-nom` par ton nom d'utilisateur si tu gardes le filesystem server. Pour le Meta Ads MCP, le bloc est identique sur Mac et Windows et l'URL ne contient aucun ID de Business : c'est le flow OAuth qui se chargera de router vers le bon compte au moment de la connexion.

**Sauvegarde le fichier.** Sur Mac : CMD + S. Sur Windows : CTRL + S.

<Callout variant="tip" title="Erreur classique : le JSON mal formaté">
Le piège numéro 1 des débutants : une virgule en trop (ou manquante) dans le JSON. Un JSON invalide fait planter le chargement du serveur MCP sans message d'erreur clair. Si le marteau n'apparaît pas après redémarrage, copie ton JSON dans jsonlint.com (outil de validation JSON en ligne gratuit). Il te dira exactement où est l'erreur.
</Callout>

<SectionHeader number="06" title="Étape 4 : redémarrer Claude Desktop" eyebrow="Redémarrage" />

Le fichier de configuration n'est lu qu'au démarrage de Claude Desktop. Il faut faire un vrai redémarrage, pas juste fermer et rouvrir la fenêtre.

**Sur Mac :**
1. Dans la barre de menus, clique sur **Claude** puis **Quit Claude**
2. Ou utilise CMD + Q

**Sur Windows :**
1. Clic droit sur l'icône Claude dans la barre des tâches
2. Clique sur **Exit** ou **Fermer**
3. Rouvre Claude depuis le Bureau ou le menu Démarrer

**Ce que tu dois voir après redémarrage.**

Dans la zone de saisie de chat, en bas à droite, une icône apparaît. Sur la documentation officielle Anthropic, c'est décrite comme une icône de marteau (hammer icon). Si tu la vois, le serveur MCP est actif. Clique dessus pour voir les tools disponibles et confirmer que `meta-ads` est bien dans la liste.

Si l'icône n'apparaît pas, passe à la section dépannage plus bas.

<SectionHeader number="07" title="Étape 5 : autoriser ton compte Meta" eyebrow="Authentification" />

Le MCP Meta est actif, mais il n'a pas encore accès à ton compte. Tu dois lui donner les permissions via un flow OAuth — un système d'autorisation standard utilisé partout sur internet (le même bouton qu'utilise "Se connecter avec Google" ou "Se connecter avec Facebook" sur les sites tiers). Tu ne fournis aucun mot de passe à Claude : tu autorises Meta à laisser Claude consulter ton compte, et tu peux révoquer cet accès à tout moment depuis tes paramètres Meta Business.

**Comment déclencher l'autorisation — la voie la plus simple :**

1. Clique sur l'icône **marteau** en bas à droite de la zone de chat Claude.
2. Tu vois la liste des MCP connectés, dont `meta-ads`. À côté, un bouton ou une mention "Non connecté" / "Connecter".
3. Clique sur **Connecter** (ou "Sign in", selon ta version Claude).
4. Une fenêtre Meta s'ouvre dans ton navigateur → tu suis le flow décrit ci-dessous.

**Voie alternative — par requête à Claude :**

Si tu ne vois pas le bouton de connexion dans l'interface marteau, tape dans la zone de chat : *"Connecte mon compte Meta Ads, je veux interroger mes campagnes."* Claude va te répondre qu'il a besoin d'une autorisation et te proposer un lien cliquable qui ouvre la fenêtre Meta.

**Important : quel compte Facebook utiliser ?**

Utilise le compte Facebook qui a le rôle **Admin** sur le Business Manager qui contient ton compte publicitaire. Si tu as plusieurs Business Managers, tu peux autoriser ceux que tu veux. Si tu gères des comptes clients, autorise les comptes pour lesquels tu as les droits.

<Callout variant="tip" title="Tu ne vois pas ton compte publicitaire dans la liste ?">
Deux raisons possibles : (1) ton compte Facebook n'a pas le rôle Admin sur le Business Manager concerné (vérifie dans business.facebook.com, onglet Paramètres, puis Personnes). (2) ton compte publicitaire est en restriction ou en revue. Les comptes en état non-actif n'apparaissent pas dans le flow d'autorisation MCP.

<SectionHeader number="08" title="Étape 6 : tester que tout fonctionne" eyebrow="Vérification" />

L'installation est censée être terminée. Voici comment vérifier que ce n'est pas juste une configuration silencieuse qui ne fait rien.

**Test 1 : le reporting de base.**

Tape dans Claude :

> "Donne-moi le ROAS total de mes campagnes Meta des 7 derniers jours."

Si Claude retourne un chiffre (même approximatif), compare-le avec ce que tu vois dans Ads Manager. Si les valeurs sont cohérentes, le MCP est opérationnel.

Tape :

> "Liste-moi toutes mes campagnes Meta actives avec leur statut et leur budget quotidien."

Si Claude retourne une liste de campagnes avec des noms, statuts et budgets qui correspondent à ce que tu as dans Ads Manager, tout est bon.

**Test 3 : une requête plus précise.**

Tape :

> "Quels sont les 3 créatifs avec le meilleur CTR sur mon compte Meta ces 14 derniers jours ?"

Ce test vérifie que le MCP sait aller chercher des données granulaires (niveau ad), pas seulement du reporting de haut niveau.

<SectionHeader number="09" title="Dépannage : les 5 problèmes les plus fréquents" eyebrow="Si ça ne marche pas" />

Tu as suivi toutes les étapes mais quelque chose coince. Voici les 5 problèmes les plus fréquents et leurs solutions.

**Problème 1 : l'icône marteau n'apparaît pas après redémarrage.**

Cause la plus probable : le JSON est mal formaté. Solution :
1. Copie le contenu de `claude_desktop_config.json`
2. Colle-le dans jsonlint.com
3. Corrige l'erreur identifiée (virgule en trop, guillemets mal échappés, accolade non fermée)
4. Sauvegarde et redémarre

Si le JSON est valide et l'icône toujours absente : ta version de Claude Desktop est peut-être trop ancienne pour gérer les connecteurs MCP remote. Clique sur le menu **Claude** puis **Check for Updates...**. Une fois sur la dernière version, relance Claude.

**Problème 2 : Claude dit qu'il n'a pas accès à Meta.**

Cause : l'autorisation OAuth n'a pas abouti ou a expiré. Solution : demande à Claude "Reconnecte-moi à Meta" ou va dans les paramètres du connecteur depuis l'icône marteau et clique sur "Reconnecter".

**Problème 3 : les données retournées ne correspondent pas à ce que je vois dans Ads Manager.**

Cause la plus probable : plage de dates ou compte publicitaire différent. Précise dans ta requête :
- L'ID du compte publicitaire (format `act_XXXXXXXXX`, visible dans Ads Manager)
- La plage de dates exacte

Exemple : "Donne-moi le ROAS du compte act_123456789 du 1er au 10 mai 2026."

**Problème 4 : Claude retourne une erreur API Meta.**

Causes possibles : ton compte publicitaire est en restriction, ou les permissions accordées au MCP sont insuffisantes. Vérifie l'état de ton compte dans Ads Manager. Si le compte est actif, essaie de révoquer et réaccorder les permissions depuis les Paramètres de l'application Business.

**Problème 5 : tout fonctionne mais les réponses de Claude sont lentes.**

Le MCP doit faire des appels API à Meta à chaque requête. Sur un compte avec beaucoup de campagnes et d'historique, une requête type "analyse mes 30 derniers jours" peut prendre 15 à 30 secondes. C'est normal. Pour accélérer, restreins la plage de dates et le nombre de campagnes dans ta requête.

<Callout variant="tip" title="Consulter les logs pour debugger">
Si Claude Desktop charge ton MCP sans succès et que tu ne comprends pas pourquoi, les logs sont tes meilleurs alliés. Sur Mac : ouvre le Terminal et tape `tail -n 20 -f ~/Library/Logs/Claude/mcp*.log`. Sur Windows : ouvre `%APPDATA%\Claude\logs\mcp*.log` dans le Bloc-notes. Les erreurs sont souvent explicites : "Connection failed" = serveur Meta injoignable ou URL mal recopiée. "Auth error" ou "Unauthorized" = l'OAuth Meta n'a pas abouti ou la session a expiré, relance le flow de connexion depuis Claude.
</Callout>

<SectionHeader number="10" title="Tes 5 premières requêtes utiles une fois connecté" eyebrow="Premiers pas" />

Tu es connecté. Le MCP fonctionne. Par où commencer ?

Voici les 5 requêtes qui te donnent de la valeur immédiate, sans risque (toutes en lecture, aucune écriture sur ton compte).

**Requête 1 : ton dashboard quotidien en 10 secondes.**

> "Donne-moi un résumé de la journée d'hier : spend total, ROAS, CPM, nombre de campagnes actives. Classe par spend décroissant."

Tu obtiens en une requête ce qui te prenait 3 onglets à compiler.

**Requête 2 : identifier les créatifs qui tirent le compte vers le bas.**

> "Quels sont les 5 créatifs avec le CPA le plus élevé sur les 14 derniers jours, et quelle est leur dépense totale ?"

Ce n'est pas de l'optimisation (tu ne touches à rien, tu informes ta décision). C'est de la lecture. Zéro risque.

**Requête 3 : détecter les ad sets en learning phase.**

> "Liste-moi tous les ad sets qui sont actuellement en learning phase sur mon compte."

Tu vois d'un coup quels ad sets sont instables et évites de les toucher. Ça prend 20 secondes vs 5 minutes de navigation Ads Manager.

**Requête 4 : un brief pour ta semaine.**

> "Résume les performances de cette semaine vs la semaine dernière. Qu'est-ce qui a bougé significativement (à la hausse ou à la baisse) ?"

L'agent compare automatiquement et te ressort les deltas. C'est la base du reporting hebdomadaire.

**Requête 5 : état de santé du Pixel.**

> "Donne-moi un diagnostic de l'état de mon Pixel Meta : est-ce qu'il reçoit bien les événements d'achat ? Y a-t-il des erreurs ?"

Le MCP couvre les signal diagnostics. C'est utile si tu doutes de la qualité de ta mesure.

<Callout variant="tip" title="Règle d'or pour débuter : lecture uniquement pendant 2 semaines">
Les actions en écriture du MCP Meta (créer une campagne, modifier un budget, pause un créatif) sont puissantes mais demandent un encadrement précis de l'agent pour éviter les resets de learning phase. Pour tes 2 premières semaines, limite-toi aux requêtes de lecture comme celles ci-dessus. Tu construis ta confiance dans l'outil et ta compréhension de ses réponses avant d'autoriser des actions. Pour les garde-fous avancés et les system prompts sécurisés, c'est l'objet de l'article dédié aux guardrails Claude Code + Meta.
</Callout>

<SectionHeader number="12" title="La suite logique : aller plus loin avec ton MCP Meta" eyebrow="Prochaine étape" />

Tu as maintenant le MCP Meta opérationnel. Voici les 3 prochaines étapes logiques selon ton niveau d'ambition.

**Niveau 1 : consolider l'usage en lecture (semaines 1-2).**

Utilise le MCP uniquement pour du reporting. Construis tes 3-4 requêtes quotidiennes habituelles (dashboard matin, analyse créatifs, vérification learning phase, brief hebdo). L'objectif : comprendre ce que l'agent retourne, calibrer tes formulations, gagner en confiance.

**Niveau 2 : activer les actions en écriture avec garde-fous (semaine 3+).**

Une fois que tu fais confiance aux réponses en lecture, tu peux activer des actions d'écriture simples (créer une campagne depuis un brief, ajouter un créatif, dupliquer un ad set). Avec un system prompt structuré qui lui interdit de modifier des ad sets actifs sans confirmation humaine. Le guide complet des guardrails est dans l'article dédié sur <a href="/blog/claude-code-meta-ads-system-prompts-guardrails-2026/">les system prompts et garde-fous pour piloter Meta sans risque</a>.

**Niveau 3 : stack multi-MCP (progressif).**

Si tu pilotes aussi du Google Ads ou du Amazon Ads, tu peux brancher leurs MCP servers dans le même Claude Desktop. Un seul agent, plusieurs plateformes. La requête type devient : "Compare mon CPA Meta vs Google Ads cette semaine." L'article comparatif des MCP servers ad tech couvre l'ordre de priorité selon ton profil DNVB.

Just be sure to remind your agent about the learning phase.

Ce conseil d'un Client Partner Meta résume le seul vrai garde-fou à avoir en tête. Le MCP officiel est Meta-authentifié, conforme aux Marketing API Terms.
